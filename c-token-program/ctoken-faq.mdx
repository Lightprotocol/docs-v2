#### What is light-token?
Light-token is like SPL token stored more efficiently to reduce account creation cost.
Users receive and send the same tokens.

| Creation Cost        | SPL               | Light            |
|:---------------------|:------------------|:-------------------|
| **Mint Account**    | ~1,500,000 lamports        | **15,000** lamports     |
| **Token Account**    | ~2,000,000 lamports | ~**17,000** lamports |

#### Is light-token live on mainnet & audited?
Light-token is currently deployed on devnet and undergoing security audits. 
<br/>Mainnet ETA: End of Q1 2026.

#### Do I need to change my client code significantly?
No. The `light-token-sdk` follows similar patterns to SPL.

<Accordion title="Can c-token accounts hold SPL tokens?">
Yes. c-token accounts can hold tokens from c-mints, SPL mints, or Token 2022 mints.

SPL tokens can be deposited into c-token accounts and withdrawn back to SPL token accounts via the `transferInterface` method.
</Accordion>

<Accordion title="Do c-token accounts require rent?">

<Accordion title="What happens if my c-token account runs out of rent?">
The account is automatically compressed.
Your tokens are preserved as a compressed token account (rent-free). 
The account is automatically decompressed when someone interacts with it again.
</Accordion>

<Accordion title="What is the difference between c-token and compressed token?">
- **c-token**: Solana account that holds token balances of c-mints, SPL or Token 22 mints.
- **Compressed token**: Compressed account storing token data. Rent-free, for storage and distribution.
</Accordion>

<Accordion title="How do I query c-token and compressed token balances?">
Standard Solana RPC works for c-token accounts (Solana accounts).
</Accordion>
