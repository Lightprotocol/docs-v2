Creating a token account on Solana costs ~0.002 SOL in rent-exemption. For applications handling thousands of users, this adds up.

c-Token reduces account creation cost to ~0.00017 SOL.

## What is c-Token?

c-Token extends Solana's token standards with compressed storage. It introduces two primitives:

- **c-Mint**: Compressed account representing a token. Rent-free.
- **c-Token**: Solana account that holds token balances. Uses sponsored rent instead of rent-exemption.

|  Creation Cost       | SPL               | c-Token            |
|:---------------------|:------------------|:-------------------|
| **Mint Account**     | ~1,500,000 lamports | **15,000** lamports |
| **Token Account**    | ~2,000,000 lamports | **17,000** lamports |

## Do I need to rewrite my code?

No. The SDK follows similar patterns to SPL.

## What about existing SPL tokens?

c-Token accounts can hold tokens from c-Mints, SPL mints, or Token 2022 mints. You can convert SPL tokens to c-Tokens and back via `transferInterface`.

## The Rent Model

c-Token accounts don't require upfront rent-exemption. Instead, rent is paid over time when accounts are accessed:

When rent runs out, the account is automatically compressed. Your tokens are preserved as a compressed token account (rent-free). The account decompresses automatically when you interact with it again.

## c-Token vs Compressed Tokens

A common point of confusion:

| | c-Token | Compressed Token |
|:--|:--|:--|
| **Storage** | Solana account | Compressed account |
| **Rent** | Sponsored (pay-as-you-go) | None |
| **Use case** | Active wallets, trading | Distribution, storage |
| **RPC** | Standard Solana RPC | ZK Compression RPC |

Use c-Token for active token accounts. Use compressed tokens for distribution and long-term storage.
