~~~~text
## Extension Overview

### Fully Compressed Extensions
These extensions work with both light-token and compressed token accounts:
- **TokenMetadata**: Add name, symbol, URI, and custom key-value metadata
- **InterestBearingConfig**: Configure interest-bearing tokens
- **GroupPointer/GroupMemberPointer**: Token grouping capabilities
- **TokenGroup/TokenGroupMember**: Group membership management

### Light-Token Only Extensions
These extensions require an on-chain SPL mint and work only with light-token accounts:
- **MintCloseAuthority**: Authority to close the mint
- **TransferFeeConfig**: Transfer fees (must be zero for compatibility)
- **DefaultAccountState**: Default state for new accounts
- **PermanentDelegate**: Permanent delegation authority
- **TransferHook**: Transfer hooks (program_id must be nil)
- **Pausable**: Ability to pause transfers
- **Confidential extensions**: Initialized but not enabled

### Phase 3: Implement mint creation with extensions

See Code Reference below for Rust implementation.

1. **RPC Setup**: Connect to ZK Compression RPC endpoint (Helius, Triton, or localnet)
   - Use `LightClientConfig::devnet()` or `LightClientConfig::mainnet()` with RPC URL and API key
   - Use `LightClient::new_with_retry()` to initialize client
   - For localnet: Configure with localhost URL

2. **Generate Mint Signer**: Create a new keypair for the mint
   - Use `Keypair::new()` to create mint signer
   - This keypair signs the mint creation transaction

3. **Derive Addresses**: Calculate mint PDA and compression address
   - Use `find_cmint_address(&mint_signer.pubkey())` to get mint PDA
   - Use `derive_cmint_compressed_address(&mint_signer.pubkey(), &address_tree.tree)` to get compression address
   - The compression address is stored in the address tree

4. **Get Tree Info**: Fetch address tree and state tree information
   - Use `rpc.get_address_tree_v2()` for address tree
   - Use `rpc.get_latest_active_state_trees()` and `get_random_state_tree_info()` for state tree
   - Select output queue pubkey from state tree info

5. **Fetch Validity Proof**: Get proof that mint address does not exist yet
   - Use `rpc.get_validity_proof(vec![], vec![AddressWithTree], None)`
   - Pass empty hash array and compression address with tree
   - Returns proof and root indices for instruction

6. **Build Extension Parameters**: Configure extensions based on requirements
   - **TokenMetadata**: Add name, symbol, URI, and custom metadata
   - **Other extensions**: Configure based on use case
   - Each extension has its own instruction data type
   - Multiple extensions can be combined in the `extensions` vector

7. **Build Params**: Configure mint parameters with extensions
   - Create `CreateCMintParams` with decimals, authorities, proof, and addresses
   - Add extensions via `extensions: Some(vec![ExtensionInstructionData::...])`
   - Extensions are optional (can be `None`)

8. **Build Instruction**: Create instruction with all parameters
   - Call `CreateCMint::new(params, mint_signer, payer, address_tree, output_queue)`
   - Call `.instruction()` to build the instruction
   - Instruction includes all required system accounts

9. **Send Transaction**: Sign and submit transaction to network
   - Use `rpc.create_and_send_transaction(&[instruction], &payer.pubkey(), &[payer, &mint_signer])`
   - Transaction requires both payer and mint signer signatures
   - Returns transaction signature on success

### Complete Mint Creation with TokenMetadata Extension

```rust
use light_client::indexer::{AddressWithTree, Indexer};
use light_client::rpc::{LightClient, LightClientConfig, Rpc};
use light_token_sdk::token::{CreateCMint, CreateCMintParams};
use light_token_interface::instructions::extensions::token_metadata::TokenMetadataInstructionData;
use light_token_interface::instructions::extensions::ExtensionInstructionData;
use light_token_interface::state::AdditionalMetadata;
use serde_json;
use solana_sdk::{bs58, pubkey::Pubkey, signature::Keypair, signer::Signer};
use std::convert::TryFrom;
use std::env;
use std::fs;

#[tokio::test(flavor = "multi_thread")]
async fn test_create_mint_with_metadata() {
    dotenvy::dotenv().ok();

    let keypair_path = env::var("KEYPAIR_PATH")
        .unwrap_or_else(|_| format!("{}/.config/solana/id.json", env::var("HOME").unwrap()));
    let payer = load_keypair(&keypair_path).expect("Failed to load keypair");

    let api_key = env::var("api_key")
        .expect("api_key environment variable must be set");

    let config = LightClientConfig::devnet(
        Some("https://devnet.helius-rpc.com".to_string()),
        Some(api_key),
    );
    let mut rpc = LightClient::new_with_retry(config, None)
        .await
        .expect("Failed to initialize LightClient");

    // Create mint with TokenMetadata extension
    let (mint_pda, compression_address) = create_mint_with_metadata(&mut rpc, &payer).await;

    println!("\n=== Created Mint with TokenMetadata Extension ===");
    println!("Mint PDA: {}", mint_pda);
    println!("Compression Address: {}", bs58::encode(compression_address).into_string());
}

pub async fn create_mint_with_metadata<R: Rpc + Indexer>(
    rpc: &mut R,
    payer: &Keypair,
) -> (Pubkey, [u8; 32]) {
    let mint_signer = Keypair::new();
    let address_tree = rpc.get_address_tree_v2();

    // Fetch active state trees
    let _ = rpc.get_latest_active_state_trees().await;
    let output_pubkey = match rpc
        .get_random_state_tree_info()
        .ok()
        .or_else(|| rpc.get_random_state_tree_info_v1().ok())
    {
        Some(info) => info
            .get_output_pubkey()
            .expect("Invalid state tree type for output"),
        None => {
            let queues = rpc
                .indexer_mut()
                .expect("IndexerNotInitialized")
                .get_queue_info(None)
                .await
                .expect("Failed to fetch queue info")
                .value
                .queues;
            queues
                .get(0)
                .map(|q| q.queue)
                .expect("NoStateTreesAvailable: no active state trees returned")
        }
    };

    // Derive addresses
    let compression_address = light_token_sdk::token::derive_cmint_compressed_address(
        &mint_signer.pubkey(),
        &address_tree.tree,
    );
    let mint_pda = light_token_sdk::token::find_cmint_address(&mint_signer.pubkey()).0;

    // Get validity proof
    let rpc_result = rpc
        .get_validity_proof(
            vec![],
            vec![AddressWithTree {
                address: compression_address,
                tree: address_tree.tree,
            }],
            None,
        )
        .await
        .unwrap()
        .value;

    // Build params with TokenMetadata extension
    let params = CreateCMintParams {
        decimals: 9,
        address_merkle_tree_root_index: rpc_result.addresses[0].root_index,
        mint_authority: payer.pubkey(),
        proof: rpc_result.proof.0.unwrap(),
        compression_address,
        mint: mint_pda,
        freeze_authority: None,
        extensions: Some(vec![ExtensionInstructionData::TokenMetadata(
            TokenMetadataInstructionData {
                update_authority: Some(payer.pubkey().to_bytes().into()),
                name: b"Example Token".to_vec(),
                symbol: b"EXT".to_vec(),
                uri: b"https://example.com/metadata.json".to_vec(),
                additional_metadata: Some(vec![
                    AdditionalMetadata {
                        key: b"category".to_vec(),
                        value: b"utility".to_vec(),
                    },
                    AdditionalMetadata {
                        key: b"type".to_vec(),
                        value: b"compressed".to_vec(),
                    },
                ]),
            },
        )]),
    };

    // Create and send instruction
    let create_cmint = CreateCMint::new(
        params,
        mint_signer.pubkey(),
        payer.pubkey(),
        address_tree.tree,
        output_pubkey,
    );
    let instruction = create_cmint.instruction().unwrap();

    rpc.create_and_send_transaction(&[instruction], &payer.pubkey(), &[payer, &mint_signer])
        .await
        .unwrap();

    (mint_pda, compression_address)
}

fn load_keypair(path: &str) -> Result<Keypair, Box<dyn std::error::Error>> {
    let path = if path.starts_with("~") {
        path.replace("~", &env::var("HOME").unwrap_or_default())
    } else {
        path.to_string()
    };
    let file = fs::read_to_string(&path)?;
    let bytes: Vec<u8> = serde_json::from_str(&file)?;
    Ok(Keypair::try_from(&bytes[..])?)
}

### TokenMetadata Configuration

TokenMetadata is the most commonly used extension for compressed tokens:

```rust
use light_token_interface::instructions::extensions::token_metadata::TokenMetadataInstructionData;
use light_token_interface::instructions::extensions::ExtensionInstructionData;
use light_token_interface::state::AdditionalMetadata;

let metadata = ExtensionInstructionData::TokenMetadata(
    TokenMetadataInstructionData {
        update_authority: Some(payer.pubkey().to_bytes().into()),
        name: b"My Token".to_vec(),                    // required, max 32 bytes
        symbol: b"MTK".to_vec(),                       // required, max 10 bytes
        uri: b"https://example.com/metadata.json".to_vec(),  // required, max 200 bytes
        additional_metadata: Some(vec![
            AdditionalMetadata {
                key: b"custom_field".to_vec(),         // max 32 bytes
                value: b"custom_value".to_vec(),       // max 200 bytes
            },
        ]),
    },
);
```

**Important TokenMetadata notes:**
- `name`, `symbol`, and `uri` are required fields
- `update_authority` can update standard fields and modify/remove existing additional_metadata keys
- New additional_metadata keys cannot be added after mint creation
- Only existing keys can be modified or removed

### Extension Compatibility Rules

**For compressed token accounts** (fully compressed):
- Only use extensions with "Compressed Token: ✓" in the table
- These include: TokenMetadata, MetadataPointer, InterestBearingConfig, GroupPointer, GroupMemberPointer, TokenGroup, TokenGroupMember

**For light-token accounts** (on-chain SPL mint):
- All extensions marked "light-token: ✓" are available
- Some have restrictions (e.g., TransferFeeConfig fees must be zero)
- Confidential extensions must be initialized but not enabled

### Important Notes

1. **Extension Compatibility**: Not all Token 2022 extensions work with compressed tokens. Check the compatibility table before using an extension.

2. **Fully Compressed vs Light-Token**:
   - Fully compressed tokens (no on-chain SPL mint) support only basic extensions
   - Light-token accounts (with on-chain SPL mint) support all extensions but with some restrictions

3. **Extension Restrictions**:
   - TransferFeeConfig: Fees must be zero for compatibility
   - TransferHook: program_id must be nil
   - Confidential extensions: Must be initialized but not enabled

4. **Metadata Updates**:
   - Standard fields (`name`, `symbol`, `uri`) can be updated by `update_authority`
   - For `additional_metadata`: Only existing keys can be modified or removed
   - New metadata keys cannot be added after mint creation

5. **Mint Authority**: The mint authority must sign the mint creation transaction

6. **Rent Exemption**: Light mints are on-chain accounts like SPL mints, but the light token program sponsors the rent-exemption cost (15,000 lamports)

7. **Compression Address**: The compression address is derived deterministically from the mint signer and address tree

8. **Validity Proof**: The validity proof proves that the compression address does not exist yet in the address tree
~~~~
